Server News & Updates

Server Changelog: July 12th

Hello everyone,

A new server patch has been deployed, incorporating the latest updates and stability improvements from the latest commits to the master branch. Please see the summary of changes below.

  • Initial Guild Implementation: Guilds have been partially implemented. You can now create a guild, which will appear as a cosmetic title above your character. Full guild management (inviting members, changing banners, etc.) is not yet available. Please note there is no duplication check, so friends can create guilds with the same name to appear as if you're in the same guild. yay!
  • Game Master System: Game Masters can now control system content. For now, this includes the ability to kick players from ranches.
  • General Improvements: Numerous other bug fixes and stability improvements have been implemented.

Important Notice Regarding Data Wipes

Unfortunately, to implement these new features and fixes, a full data wipe was necessary. We apologize for the inconvenience and appreciate your understanding as we work on improving and updating the server.

Disclaimer: This server is a test environment. As such, it is not stable and users may encounter crashes, bugs, and other unexpected issues.

Thank you for your understanding.

Server Changelog: July 8th

Hello everyone,

A new server patch has been deployed, incorporating the latest updates and stability improvements from the latest commits to the master branch. Please see the summary of changes below.

  • Ranch Ownership Implemented: The ranch ownership system has been implemented. Previously, while your ranch displayed your name, you did not truly "own" it in the backend. This fundamental change properly assigns the ranch to your account and is crucial for enabling future ranch-related features.
  • Ranch Management Unlocked: With the new ownership system, the Ranch Management tab is now functional, allowing you to customize and decorate your ranch with various buildings and structures. Please note that as the player level function has not been implemented yet, some structures that require a certain level cannot be built.
  • Horse Renaming Fix: Horses without a name can now be properly renamed. This applies to the initial naming prompt for the three horses you receive when creating a new character. The server now correctly handles this feature.
  • Visit Preference Fix: Corrected an issue with the //visit command where the visit preference was not resetting after use.
  • Backend YAML Support: Added a new library to support YAML configurations, improving server-side data management for future updates.
  • Data Director Fixes: Implemented several bug fixes and stability improvements to the data director.

Important Notice Regarding Data Wipes

Unfortunately, to implement these fundamental ranch changes, a full data wipe was necessary. We apologize for the inconvenience and appreciate your understanding as we work on improving and updating the server.

Disclaimer: This server is a test environment. As such, it is not stable and users may encounter crashes, bugs, and other unexpected issues.

Thank you for your understanding.

Server Changelog: July 3rd

Hello everyone,

A new server patch has been deployed, incorporating the latest updates and stability improvements from the latest commits to the master branch. Please see the summary of changes below.

  • Randomized Horses: Upon character creation, you will now be assigned three random horses. You can also get a new random horse at any time by using the command //horse randomize in the ranch chat (requires a client restart).
  • Character Voices: An issue has been resolved where character voices did not match the selected gender.
  • Emblem Customization: You can now change your emblem using the command //emblem <ID> in the ranch chat (requires a client restart). A complete list of emblem IDs can be found on SAMUELLL's website.
  • Automatic Restarts: The server is now configured to restart automatically following a crash. Please be aware that in the event of a crash, data may be lost if it was not saved prior to the incident.
  • General Fixes: This update also includes various background bug fixes and server stability improvements.

Important Notice Regarding Data Wipes

Please be advised that all player data has been wiped. This was a necessary step to accommodate ongoing development and recent server-side changes. As the project evolves, further data wipes will be required periodically to support future updates.
